Description:

Since the theme for the capstone is emotion I first taught about all the possible emotions I could work with and which one I taught was the most feasable to make happen. I taught about what I should do in my app where it to be about anger, happiness, sadness, or empathy and how to make it happen. After thinking it over for about a day I came to the realization to choose empathy. I then taught about how to do so. I remembered a VR experience called Abe in which the user does not move at all during the whole experience only listening to a robot who is about to kill you. And I tought about how I could make my app into more of an experience than the usual game most would expect. With that in mind I went about thinking of a topic whithin empathy I wanted to talk about which was the class systems that we have in place and how all of us have no power over where we initially start. To do so I decided to create a test to decide what the fate of the user should be. I then tought about the way in whuich I could implement this in my app using methods tought by in the class.

Implementation

  • I implemented sounds throughout the map which are activated as soon as the users left controller is within range. -Doable
  • There are five questions the user must answer. -Doable
  • To answer the questions I dabled on weather making it so when the user picks up an object or when a user touches an object then destroys it thus signaling the answer the user had chosen both where doable options. In the end I chose to use the later option of touching an object and then destroying it since I tought it would lead to less confusion when picking an answer. -Doable
  • I then though about the perfect setting for this test and went to unity's asset store to buy myself a decent scene. I saw two scenes that looked promising. One was set in the woods and was more of a natural environment and the other one was ruins in the desert at night. In the end I ended up choosing a environment that was set in desert ruins at night to give the app a mystical feeling to it.I also tought that it would fit in nicely with my test of deciding onces class as ruins are a chaotic mess and since people do not have the ability to choose their position as it is random thus chaotic it would fit perfectly to the message I am trying to send to the user with this app. -Doable
  • I then taught about how to layout the map. I looked through the assets included in the Desert Scene I had bought from unity and found that there was a platform asset that would be perfect for the app. -Doable
  • Then I thought about how to implement the sounds in the app. I thought about it for a while and came up with only one possible solution which I tought would work which was to have empty objects which have colliders placed at the entrance of the platform asset.These empty object colliders would be set to trigger so the user can interact with them and thus enable the question to be read out to the user via AudioClip attached to empty objects which I tought was Doable. -Doable
  • I then thought about all the questions I will have for the app and how to decide the class the user will receive. I chose to have only one question determine the the users fate while the other are there just for filler.In order to do the question I have user touch the square object with the left controller which will trigger all the squares to be destroyed by using the function OnTriggerEnter. -Doable
  • I then had a boolean keep track of the question if all where true then I would make a boolean true in order to receive The users fate. -Doable
  • You do have to do all the questions in order to receive your fate and I also decided to implement points which are being tracked by an int. -Doable
  • The user receives the points earned after finishing all 5 stations by way of AudioClip I stuck to an object close to the end. -Doable
  • Then after this is done I decided to implement a teleportation system. First I was going to use the teleportation system taught in the class but then I decided to use an external library for teleportaion called ViveTeleporter which can be found here https://github.com/Flafla2/Vive-Teleporter. -Doable
  • Then I decided on how to give the user their fate. I decided to create a rude object on Blender in order to meet an achievement which I called the Oracle. -Doable
  • Oracle tells the user their fate after finishing the questions and getting close to the Oracle by way of an emty object with a collider and a AudioClip inside of 3 empty objects near the Oracle. Each empty object containing A different AudioClip for the users class based on the main question answered by the student. -Doable
